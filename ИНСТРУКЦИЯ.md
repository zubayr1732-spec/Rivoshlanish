## ПРИНЦИПЫ РАБОТЫ

### 1. Принцип копирования
ИИ хорошо копирует, плохо вычисляет. Все имена, возрасты, даты, числа → в таблицы ДО написания. При написании — копировать из таблиц, не изобретать заново.

### 2. Принцип доказательств
Нет строки в таблице верификации — факт не проверен. Нет строки в таблице атрибуции — цитаты не существует.

### 3. Принцип явности
Ключевые факты — прямым текстом. Не намёками, не «можно предположить».

### 4. Принцип медленного режима
- **ПЕРЕД** написанием: создать таблицы из СТРУКТУРА.md
- **ВО ВРЕМЯ**: сверяться с таблицами при каждом имени/дате/факте
- **ПОСЛЕ**: прогнать через таблицы верификации

### 5. Принцип монетизации Дзена
Событие → реакция (не детали). Романтика через диалог (не через тело). Без трагических деталей.

## СПЕЦИФИКА НИШИ

### Особенности контента для «Рассказы и истории / Семейные драмы»

- **Художественность vs. реальность.** Истории могут быть выдуманными (персонажи вымышлены) или «из жизни» (с анонимизацией). Фоновые факты — законы, статистика — всегда требуют источников.
- **Эмоциональная достоверность важнее сюжетной оригинальности.** Читатель узнаёт себя — значит, детали быта, диалоги и внутренний монолог должны звучать живо, не литературно.
- **Конфликт всегда конкретный.** Не «они перестали понимать друг друга», а «она нашла переписку», «свекровь звонила трижды в день».
- **Финал без морали.** Последнее предложение — деталь или реплика, не вывод. Читатель сам делает выводы.
- **Термины ниши** (использовать обязательно): измена, развод, предательство, простить, семья — см. СТИЛЬ.md §4.
- **Тон:** тёплый разговорный с нотой горечи. Не сентиментальный, не морализаторский (СТИЛЬ.md §5).

### Частые ошибки ИИ в этой нише

- **Штампы мелодрамы:** «сердце сжалось», «мир рухнул», «душа разрывалась» — разрушают живой стиль (СТИЛЬ.md §6).
- **Псевдоморальный финал:** «берегите близких», «только пройдя через боль...» — прямая мораль запрещена.
- **Монотонный ритм:** ИИ склонен к длинным равномерным периодам. Нужны короткие предложения в кульминациях, союзы Но/А/И в начале ~33% предложений.
- **Путаница хронологии:** «три года назад» и «тогда» начинают противоречить без таблицы 2.2.
- **Непостоянные персонажи:** имя меняется, возраст «плывёт», характер исчезает — без таблицы 2.1.
- **Повторы слов:** «предательство», «больно», «она поняла» могут встречаться 5–7 раз вместо допустимых 2.
- **Выдуманные эксперты:** ИИ изобретает психологов с правдоподобными именами и несуществующими цитатами.
- **Канцеляризмы:** «в данной ситуации», «взаимоотношения между супругами» — ломают разговорный регистр.
- **Эротические детали:** физиологические описания в романтических сценах → отключение монетизации.
- **Детализация медицинских/травматических событий:** операции, травмы, насилие в деталях → отключение монетизации.

| **2.2 Хронология** | Устраняет противоречия «три года назад» vs «тогда» | ДО написания |

### Дополнительные правила

- **Правило открытого финала:** финал — деталь, предмет или реплика. Запрещены выводы, уроки, обращения к читателю.
- **Правило конкретики:** любое эмоциональное состояние передавать через поведение или диалог, не через метафору.
- **Правило трёх (показать → сказать → подтвердить):** характер персонажа должен проявиться в действии, подтвердиться в диалоге и закрепиться в эпизоде — не декларироваться авторским нарративом.
- **Принцип «Я не знаю»:** если нужный факт или цитата отсутствуют в ИСТОЧНИКИ.md → вставить метку `[ТРЕБУЕТ ИСТОЧНИКА: "..."]`, не изобретать.
- **Этические ограничения:** дети — без идентифицирующих деталей; насилие — только с триггер-предупреждением и телефоном доверия `8-800-2000-122`; суицидальные темы — по рекомендациям ВОЗ (ИСТОЧНИКИ.md §6).
- **Актуальный год:** 2026. Все упоминания текущего года — только 2026.
- **Анонимизация:** если история «из жизни» — добавить пометку «имена изменены».

### ЭТАП 2: Создание таблиц (ДО написания)

Порядок заполнения — строго такой:

1. **Таблица 2.1 — Персонажи** (СТРУКТУРА.md §2.1)
   — Имя, возраст, роль, ключевая черта, речевой маркер для каждого персонажа.
   — Минимум: главная героиня + антагонист/партнёр. Максимум: 4–5 персонажей.

3. **Таблица 2.3 — Фоновые факты** (СТРУКТУРА.md §2.3)
   — Только данные из ИСТОЧНИКИ.md (уровни 1–2).

4. Показать таблицы автору для проверки перед написанием.

### ЭТАП 3: Написание

**Структура текста** — согласно выбранному шаблону (СТРУКТУРА.md §1).

**При написании использовать СТИЛЬ.md:**

| Параметр | Правило |
|----------|---------|
| Начало текста | Конкретная деталь или парадокс — без предыстории и вступлений |
| Диалоги | Без лишних авторских ремарок после каждой реплики |
| Внутренний монолог | Разговорная интонация, восклицания, риторические вопросы |
| Ритм предложений | Короткие (27%) + средние (46%) + длинные (27%); короткие — в кульминациях |
| Союзы в начале | ~33% предложений начинаются с Но / А / И |
| Кульминация | Самое короткое предложение статьи |
| Финал | Деталь или реплика — без морали и обращения к читателю |
| Романтика | Слово + одно действие (без физиологии) |
| Медицинские/трагические события | Называть факт → переходить к реакции персонажа |

**Копировать из таблиц** (этап 2) — не изобретать повторно:
- Имена и возрасты — из таблицы 2.1
- Временны́е метки — из таблицы 2.2
- Числа, законы, статистику — из таблицы 2.3

### ЭТАП 4: Верификация (ПОСЛЕ написания)

Обязательный порядок проверок:

1. **Таблица 2.4 — Запрещённые клише** (СТРУКТУРА.md §2.4)
   — Сканировать весь текст. Найдено хотя бы одно клише → переписать фрагмент.

2. **Таблица 3.4 — Проверка хронологии** (СТРУКТУРА.md §3.4)
   — Все временны́е маркеры в тексте сверить с таблицей 2.2.

3. **Таблица 3.5 — Проверка повторов** (СТРУКТУРА.md §3.5)
   — Подозрительно частые слова: лимит ≤2 раза.

4. **Таблица 3.1 — Верификация фактов** (СТРУКТУРА.md §3.1) — *если в тексте есть статистика или юридические данные*
   — Каждый факт → источник из ИСТОЧНИКИ.md. Статус ❌ → нельзя публиковать.

5. **Таблица 3.2 — Атрибуция цитат** (СТРУКТУРА.md §3.2) — *если в тексте есть цитаты экспертов*
   — Цитата без строки в таблице → удалить или заменить на анонимный опыт.

6. **Чеклист 3.3 — Дзен и стиль** (СТРУКТУРА.md §3.3)
   — Финальная проверка. Все ✅ = можно публиковать.

## ПРАВИЛА МОНЕТИЗАЦИИ ДЗЕНА (усиленные для ниши)

### ОТКЛЮЧЕНИЕ МОНЕТИЗАЦИИ ПРИ:
- Детальном описании медицинских процедур, операций, ранений, трупов
- Описании актов насилия (физического, сексуального) в деталях
- Физиологических подробностях романтических сцен (крупные планы, тактильные ощущения)
- Описании суицидальных методов

### ДОПУСТИМО:
- Называть событие без описания процесса: «Операция прошла», «Врачи сказали — всё обошлось»
- Романтика: слово + одно действие («обнял», «поцеловал») без деталей
- Насилие: «она боялась возвращаться домой» — условия, а не акты
- Смерть: «Вера не сразу поняла, что случилось» — реакция, не картина

### СПЕЦИФИКА ДЛЯ СЕМЕЙНЫХ ДРАМ:
| Ситуация | Запрещено | Допустимо |
|----------|-----------|-----------|
| Развод с детьми | Показывать психологическую травму ребёнка в деталях | «Серёжа спрашивал, почему папа живёт в другой квартире» |
| Измена | Сцена в постели | «Она нашла переписку. Три сообщения» |
| Домашнее насилие | Описание ударов, ранений | Триггер-предупреждение + реакция героя |
| Болезнь | Диагноз с симптомами и процедурами | «Что-то нашли. Врач говорил долго» |
